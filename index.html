THE GOLDEN PROMPT FORMULA

Structure every prompt like this:

[ROLE] You are a [specific expert/persona].
[CONTEXT] Here's the situation: [relevant background].
[TASK] Your job is to [specific action verb] [specific output].
[FORMAT] Format the response as [bullet list / table / paragraph / code].

EXAMPLE:
"You are an expert futures trading coach. I'm a beginner who just learned what a call option is. Explain how implied volatility affects option pricing in plain English. Use a simple analogy and keep it under 200 words."

Why this works:
- Role = Claude knows what expertise to channel
- Context = Claude knows who it's talking to
- Task = Clear deliverable
- Format = You get exactly what you can use

Practice this formula on everything for 1 week. Your Claude outputs will transform.
SYSTEM PROMPTS 101

A system prompt runs silently before your conversation begins. It sets Claude's:
- Persona (who it is)
- Rules (what it will/won't do)
- Goals (what it's optimizing for)
- Style (how it communicates)

BASIC SYSTEM PROMPT EXAMPLE:
"You are Marcus's personal trading assistant. You help analyze trade setups using ICT methodology. Always ask for the timeframe and current price action before giving analysis. Keep responses concise — bullet points preferred. Never give financial advice, only technical analysis."

WHERE TO SET SYSTEM PROMPTS:
- Claude.ai Projects → Project Instructions (system prompt)
- Claude API → system parameter
- Custom Skills → frontmatter + SKILL.md body

Pro tip: Save your best system prompts as templates. The same prompt, reused daily, compounds over time.
TOP PROMPT MISTAKES (AND HOW TO FIX THEM)

MISTAKE 1: Too vague
❌ "Write me something about trading."
✅ "Write a 300-word Instagram caption about why retail traders always buy the top, using the concept of FOMO. Casual tone. End with a call to action."

MISTAKE 2: No context
❌ "Fix this."
✅ "Fix this Python script — it should read a CSV and output the top 5 rows by revenue, but it's returning an error on line 12. Here's the code: [paste code]"

MISTAKE 3: Too many tasks in one prompt
❌ "Research crypto trends, write a report, make a tweet thread, and suggest a trading strategy."
✅ Break into 4 separate prompts. Feed each output into the next step.

MISTAKE 4: Skipping the format
❌ Not specifying how you want the answer → you get a wall of text
✅ Always end with: "Format as bullet points" or "Give me a table" or "Keep it under 150 words"
CHAIN PROMPTING — BUILD MULTI-STEP AI WORKFLOWS

Chain prompting = treat Claude like an assembly line, not a vending machine.

STEP-BY-STEP EXAMPLE (Content Creation Workflow):

Step 1: "Research the top 5 reasons traders fail in their first year. Give me bullet points with brief explanations."

[Take Claude's output → paste into Step 2]

Step 2: "Using these 5 reasons, write an outline for a YouTube video script. Hook, 5 main points, CTA. Keep it tight."

[Take outline → paste into Step 3]

Step 3: "Write the full script for this YouTube video. Conversational tone. 8-10 minutes when read aloud. Add transitions between sections."

WHY THIS WORKS:
- Each step is focused → better quality output
- You review and edit at each checkpoint
- Complex work gets done in 20 minutes vs 3 hours
- You maintain creative control at every stage

This is the foundation of building Skills — automating these chains.

WHAT IS A CLAUDE SKILL?

A Skill is a reusable, saved workflow that Claude can load and execute automatically when relevant.

Think of it like a playbook. Instead of re-explaining your process every time, you write it once in a SKILL.md file. Claude reads it when needed and follows those specialized instructions.

REAL EXAMPLES OF SKILLS:
- A "Trade Journal Analyzer" Skill that takes your trade history CSV and generates a performance report
- A "Content Repurposing" Skill that takes one YouTube transcript and outputs an email, tweet thread, and LinkedIn post
- A "Lead Magnet Builder" Skill that turns raw content notes into a full HTML web app

SKILL TRIGGERING:
Claude reads the skill's description and decides when to use it. Good descriptions tell Claude: what the skill does AND when to use it.

Skills = your personal AI automations. Build once, use forever.
ANATOMY OF A CLAUDE SKILL

Every Skill is a folder with this structure:

my-skill/
├── SKILL.md          ← Required. The brain of the skill.
└── (optional extras)
    ├── scripts/      ← Python/bash scripts the skill can run
    ├── references/   ← Docs Claude loads into context
    └── assets/       ← Templates, icons, files used in output

SKILL.MD STRUCTURE:

---
name: my-skill-name
description: What this skill does and WHEN to use it. Be specific.
---

# Instructions

[Your workflow instructions in plain markdown]

## Output Format

[Tell Claude exactly what the output should look like]

## Examples

[Optional: Input → Output examples]

KEY RULE: Keep SKILL.md under 500 lines. If it's longer, move sections into references/ files and point to them from SKILL.md.
WHEN TO BUILD A CLAUDE SKILL

BUILD A SKILL WHEN:
✓ You do the same multi-step process 3+ times per week
✓ You always have to re-explain your workflow to Claude from scratch
✓ You want consistent, repeatable output quality
✓ The task has objectively verifiable steps (data processing, code gen, report building)

EXAMPLES OF SKILL-WORTHY WORKFLOWS:
- Daily market summary from a list of tickers
- Converting raw notes into formatted content (blogs, tweets, emails)
- Analyzing trade data and generating a weekly performance review
- Building lead magnets from content outlines (what Marcus does!)
- Code review with specific quality standards

DON'T BUILD A SKILL FOR:
✗ One-time tasks
✗ Very simple one-step requests (Claude handles these fine without a skill)
✗ Highly subjective creative tasks where you want variety, not consistency

Rule of thumb: If you've written the same system prompt more than 3 times — turn it into a Skill.
THE SKILL DEVELOPMENT LOOP

This is the exact process for building a great Skill:

STEP 1: CAPTURE INTENT
Define what the skill should do. What triggers it? What's the output format?

STEP 2: WRITE THE DRAFT
Write your SKILL.md. Focus on instructions, output format, and examples.

STEP 3: WRITE TEST CASES
Create 2-3 realistic prompts a real user would type. Run Claude with the Skill on those prompts.

STEP 4: EVALUATE RESULTS
Look at the outputs. What worked? What was off? Grade the outputs — did they meet your criteria?

STEP 5: IMPROVE THE SKILL
Rewrite based on what you learned. Generalize the instructions. Explain the WHY behind every rule.

STEP 6: REPEAT
Run the test cases again. Compare to the previous iteration. Keep looping until the outputs consistently hit.

STEP 7: OPTIMIZE THE DESCRIPTION
Fine-tune the skill's description field so Claude knows exactly when to trigger it.

The loop is: Draft → Test → Evaluate → Improve → Repeat.

Skill Creator
A skill for creating new skills and iteratively improving them.

At a high level, the process of creating a skill goes like this:
1. Decide what you want the skill to do and roughly how it should do it
2. Write a draft of the skill
3. Create a few test prompts and run claude-with-access-to-the-skill on them
4. Help the user evaluate the results both qualitatively and quantitatively
5. While the runs happen in the background, draft some quantitative evals if there aren't any (if there are some, you can either use as is or modify if you feel something needs to change about them). Then explain them to the user
6. Use the eval-viewer/generate_review.py script to show the user the results for them to look at, and also let them look at the quantitative metrics
7. Rewrite the skill based on feedback from the user's evaluation of the results
8. Repeat until you're satisfied
9. Expand the test set and try again at larger scale

Your job when using this skill is to figure out where the user is in this process and then jump in and help them progress through these stages.

---

CAPTURING INTENT

Start by understanding the user's intent. Ask:
- What should this skill enable Claude to do?
- When should this skill trigger? (what user phrases/contexts)
- What's the expected output format?
- Should we set up test cases to verify the skill works?

---

WRITING THE SKILL.MD

Based on the user interview, fill in these components:

name: Skill identifier
description: When to trigger, what it does. This is the primary triggering mechanism — include both what the skill does AND specific contexts for when to use it. Make the description a little "pushy" to prevent undertriggering.

SKILL FILE STRUCTURE:
skill-name/
├── SKILL.md (required)
│   ├── YAML frontmatter (name, description required)
│   └── Markdown instructions
└── Bundled Resources (optional)
    ├── scripts/    - Executable code for deterministic/repetitive tasks
    ├── references/ - Docs loaded into context as needed
    └── assets/     - Files used in output (templates, icons, fonts)

PROGRESSIVE DISCLOSURE:
- Metadata (name + description) — Always in context (~100 words)
- SKILL.md body — In context whenever skill triggers (<500 lines ideal)
- Bundled resources — As needed (unlimited)

---

WRITING PATTERNS

Prefer using the imperative form in instructions.

Define output formats using exact templates:
## Report structure
ALWAYS use this exact template:
# [Title]
## Executive summary
## Key findings
## Recommendations

Include examples formatted like this:
## Commit message format
Example 1:
Input: Added user authentication with JWT tokens
Output: feat(auth): implement JWT-based authentication

---

WRITING STYLE

- Explain WHY things are important rather than using heavy-handed "MUST" statements
- Use theory of mind — try to make the skill general, not narrow to specific examples
- If you find yourself writing ALWAYS or NEVER in all caps, that's a yellow flag — reframe and explain the reasoning instead
- Write a draft, then look at it with fresh eyes and improve it

---

TEST CASES

After writing the skill draft, come up with 2-3 realistic test prompts — the kind of thing a real user would actually say. Share them with the user: "Here are a few test cases I'd like to try. Do these look right, or do you want to add more?" Then run them.

Save test cases to evals/evals.json:
{
  "skill_name": "example-skill",
  "evals": [
    {
      "id": 1,
      "prompt": "User's task prompt",
      "expected_output": "Description of expected result",
      "files": []
    }
  ]
}

---

THE IMPROVEMENT LOOP

After reviewing results:
1. Generalize from the feedback — don't overfit to specific examples
2. Keep the prompt lean — remove things that aren't pulling their weight
3. Explain the WHY behind everything you're asking the model to do
4. Look for repeated work across test cases — if all runs wrote the same helper script, bundle it
5. Apply improvements to the skill
6. Rerun all test cases into a new iteration directory
7. Wait for user review
8. Repeat until the user is happy or feedback is consistently empty

---

DESCRIPTION OPTIMIZATION

The description field is the primary mechanism that determines whether Claude invokes a skill. After creating or improving a skill, optimize the description for better triggering accuracy.

Generate 20 eval queries — a mix of should-trigger and should-not-trigger. Focus on edge cases and near-misses rather than obvious cases.

For should-trigger queries: different phrasings of the same intent, casual vs formal, implicit vs explicit requests.
For should-not-trigger queries: adjacent domains, queries that share keywords but need something different.

---

COMMUNICATING WITH THE USER

Pay attention to context cues to understand technical familiarity:
- "evaluation" and "benchmark" — borderline, OK to use
- "JSON" and "assertion" — explain these if you're not sure the user knows them
- Brief term definitions are always welcome

CORE LOOP:
1. Figure out what the skill is about
2. Draft or edit the skill
3. Run claude-with-access-to-the-skill on test prompts
4. With the user, evaluate the outputs
5. Repeat until satisfied
6. Package the final skill and return it to the user

A Claude Project gives you a persistent system prompt + uploaded knowledge base for any topic.

STEP 2: Click "Projects" in the left sidebar
STEP 3: Click "New Project"
STEP 4: Name it (e.g., "Content Creation", "Trading Assistant", "Business Operations")
STEP 5: Click "Edit project instructions" and paste this starter template:

---
You are my [role] assistant, specialized in [your niche].

My name is [your name]. My audience is [describe your audience]. My brand voice is [describe: casual/professional/bold/etc.].

When I ask for content, always:
- Match my voice and tone
- Keep it concise and actionable
- Format for [platform: Instagram / Twitter / email / etc.]
- End with a CTA when appropriate

When I ask for analysis, always:
- Give me the bottom line first
- Support with 2-3 key points
- Suggest one next action
---

STEP 6: Upload any relevant files (your brand guide, past content, data files)
STEP 7: Start chatting — Claude now has persistent context about you every session
